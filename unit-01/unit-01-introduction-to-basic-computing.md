# Part A — Foundations of Computing

## 1. Why Computing Matters for a Law Student

Law today is not only books and oral arguments. A junior associate’s first week often looks like this:

- Typing a **legal notice** in Word  
- Searching a **case citation** online  
- Receiving client instructions on **email / WhatsApp**  
- Tracking hearing dates in **Excel**  
- Preparing a **moot court** slide deck  
- Asking an **AI tool** to summarise a long judgment *(then verifying every line)*

If you treat “Computer Information” as a side subject, you will struggle in internships. If you treat it as a **professional skill**, you will look more employable from Semester 1.

### Law Office Lens — Day 1 in Chambers

Advocate Mehta’s chamber has three juniors. One knows only typing. Another can organise folders, convert PDFs, research on SCC/Manupatra-style databases, and prepare a clean notice format. Guess who gets the drafting work and client-facing responsibility first?

**Computing is not replacing advocacy. It is amplifying it.**

## 2. Definition of Computer

### 2.1 Simple Definition (Exam-Ready)

> A **computer** is an electronic device that accepts data (input), processes it according to instructions (program), stores intermediate and final results, and produces useful information (output).

### 2.2 Expanded Definition

A computer is a programmable machine that:

1. Accepts **input** (keyboard, mouse, scanner, mic, network)  
2. Performs **processing** using CPU and memory  
3. Stores data in **primary** and **secondary** storage  
4. Gives **output** (monitor, printer, speakers, files, network)  
5. Follows a set of instructions called a **program** / **software**

### 2.3 Characteristics of a Computer

| Characteristic | Meaning | Legal Workplace Example |
|---|---|---|
| Speed | Works very fast | Searching thousands of judgments in seconds |
| Accuracy | Low error if instructions are correct | Calculating court fee / interest correctly |
| Diligence | No fatigue | Overnight indexing of scanned case files |
| Storage | Huge data capacity | Archiving years of client records |
| Versatility | Many tasks | Drafting, research, billing, presentation |
| Automation | Follows programs | Auto-reminders for limitation dates |

### 2.4 What a Computer Is *Not*

- It is **not** intelligent by itself (unless software / AI is added).  
- It does **not** understand justice, fairness, or ethics — humans do.  
- It does **not** replace professional responsibility of an advocate.

### Key Definitions

- **Data:** Raw facts (e.g., “Case No. 12/2024”, “Rs. 50,000”).  
- **Information:** Processed, meaningful data (e.g., “Client X owes Rs. 50,000 as of 12 Jan 2024”).  
- **Hardware:** Physical parts you can touch.  
- **Software:** Programs and instructions.  
- **Firmware:** Software embedded in hardware (e.g., BIOS/UEFI).

### Exam Trap

Students write: “Computer is a machine that thinks.”  
Better: “Computer is an electronic device that processes data as per programmed instructions.”

### Quick Check

1. Differentiate data and information with one legal example.  
2. Write any four characteristics of a computer.

## 3. Components of a Computer System

A complete computer system = **Hardware + Software + User + Data + Procedures** (sometimes network is added as a fifth/sixth element).

For exams, focus first on the classical model:

```text
INPUT  →  PROCESS (CPU + Memory)  →  OUTPUT
              ↕
           STORAGE
```

### 3.1 Input Devices

Devices that send data into the computer.

| Device | Use in Legal Work |
|---|---|
| Keyboard | Drafting petitions, emails, agreements |
| Mouse / Trackpad | Editing, selecting text, navigating PDFs |
| Scanner | Digitising signed papers, vakalatnama, annexures |
| Microphone | Recording client meeting notes / dictation |
| Webcam | Virtual hearings, online client meetings |
| Barcode / QR scanner | File management in large offices / courts |

### 3.2 Output Devices

| Device | Use in Legal Work |
|---|---|
| Monitor | Reading drafts, researching |
| Printer | Filing hard copies, certified sets |
| Speakers / Headphones | Webinars, virtual court audio |
| Projector | Classroom / seminar / moot presentations |

### 3.3 Central Processing Unit (CPU)

The **brain** of the computer. Major parts:

1. **ALU (Arithmetic Logic Unit):** Maths and logical comparisons  
2. **CU (Control Unit):** Directs operations; fetches and decodes instructions  
3. **Registers:** Tiny ultra-fast storage inside CPU

**Law Office Lens:** When your laptop hangs while 20 PDFs and Word are open, the bottleneck is often CPU + RAM, not “Word is bad.” Closing unused apps is professional hygiene.

### 3.4 Memory / Storage

#### Primary Memory (Main Memory)

| Type | Full Form | Nature | Role |
|---|---|---|---|
| RAM | Random Access Memory | Volatile | Temporary working memory |
| ROM | Read Only Memory | Non-volatile | Boot instructions |

**Volatile** = content lost when power is off.

#### Secondary Storage

| Type | Example | Legal Use |
|---|---|---|
| HDD | Hard Disk Drive | Large archives |
| SSD | Solid State Drive | Faster modern laptops |
| USB Drive | Pen drive | Carrying drafts (use carefully; security risk) |
| Optical | CD/DVD (legacy) | Older archives |
| Cloud Storage | Drive / OneDrive / Dropbox | Remote access & backup |

### 3.5 Software Types

1. **System Software:** OS, device drivers, utilities  
2. **Application Software:** MS Word, Excel, PowerPoint, browsers, legal databases  
3. **Programming Software:** Compilers, IDEs (less common for BA-LLB core work)

### 3.6 People & Procedures

- **Users:** Advocates, clerks, students, judges’ staff  
- **Procedures:** Naming files properly, backup policy, confidentiality rules

### Law Office Lens — File Naming That Saves Careers

Bad: `finalfinal2(1).docx`  
Good: `2024-07-13_Notice_Sharma_v_State_Draft-v3.docx`

Good naming is a computing skill and a professional habit.

### Key Definitions

- **CPU:** Central Processing Unit; performs processing.  
- **RAM:** Temporary memory used while programs run.  
- **SSD/HDD:** Secondary storage for permanent files.  
- **Peripheral:** External device connected to computer (printer, scanner).

### Exam Trap

RAM is **not** permanent storage. If power fails while drafting and auto-save is off, unsaved text can be lost.

### Quick Check

1. Draw and label the IPO (Input–Process–Output) cycle.  
2. Difference between RAM and ROM.  
3. Name three input and three output devices used in a law office.

## 4. Operating Systems (OS)

### 4.1 Definition

> An **Operating System** is system software that manages computer hardware and software resources and provides common services for programs and users.

It is the **manager** between you and the machine.

### 4.2 Main Functions of an OS

1. Process management  
2. Memory management  
3. File management  
4. Device management  
5. Security and access control  
6. User interface (GUI / CLI)

### 4.3 Common Operating Systems

| OS | Type | Where You Meet It |
|---|---|---|
| Windows | Desktop/Laptop | Most Indian offices and chambers |
| macOS | Desktop/Laptop | Design-heavy / some corporate firms |
| Linux | Desktop/Server | Servers, tech teams, some labs |
| Android | Mobile | Phones/tablets |
| iOS | Mobile | iPhone / iPad |

### 4.4 GUI vs CLI

- **GUI (Graphical User Interface):** Icons, windows, menus (beginner-friendly)  
- **CLI (Command Line Interface):** Text commands (powerful; used by IT admins)

BA-LLB students mainly use GUI, but understanding CLI exists helps in tech-aware legal practice (cybercrime, e-evidence basics later).

### 4.5 Files, Folders, and Extensions

| Extension | Meaning | Legal Relevance |
|---|---|---|
| `.docx` | Word document | Notices, petitions drafts |
| `.pdf` | Portable Document Format | Filing-ready / non-easily-editable share |
| `.xlsx` | Excel workbook | Cause list tracking, billing |
| `.pptx` | PowerPoint | Moot / seminar |
| `.jpg/.png` | Images | Scanned annexures |
| `.zip` | Compressed folder | Sending multiple files |

### 4.6 Basic Security Habits (Must-Know)

1. Use a strong password / passkey.  
2. Enable screen lock in chambers.  
3. Do not leave client files on public PCs.  
4. Keep OS and antivirus updated.  
5. Be careful with unknown USB drives (malware risk).  
6. Take backups (local + cloud, if permitted by office policy).

### Law Office Lens — Confidentiality Is Also an OS Habit

Leaving a client settlement draft open on a shared desktop is not only careless — it may breach professional ethics and client trust. Locking the screen is a legal professionalism skill.

### Key Definitions

- **OS:** Interface and resource manager between user and hardware.  
- **Multitasking:** Running multiple programs seemingly at once.  
- **File:** Named collection of related data.  
- **Folder/Directory:** Container that organises files.

### Exam Trap

“Windows is an application software.”  
Correct classification: Windows is **system software** (an OS). MS Word is application software.

### Quick Check

1. Define OS and list four functions.  
2. Differentiate GUI and CLI.  
3. Why is PDF preferred for final court filings in many situations?

# Part B — Internet & Digital Communication

## 5. Internet Basics

### 5.1 Definition

> The **Internet** is a global network of interconnected computer networks that communicate using standard protocols (primarily TCP/IP).

### 5.2 Useful Related Terms

| Term | Meaning |
|---|---|
| WWW | World Wide Web — system of linked hypertext documents accessed via browsers |
| Browser | Software to access websites (Chrome, Edge, Firefox) |
| Website | Collection of related web pages |
| URL | Address of a web resource |
| ISP | Internet Service Provider |
| IP Address | Numerical identity of a device on a network |
| DNS | Converts domain names to IP addresses |
| Download / Upload | Receiving from / sending to the network |
| Cloud | Remote servers accessed over internet |

### 5.3 Internet vs WWW

- **Internet** = the infrastructure / network.  
- **WWW** = one service on the internet (others include email, FTP, messaging).

### 5.4 Legal Research on the Internet (Beginner Map)

Law students commonly use:

- Government sites (India Code, court websites, Gazette)  
- Case databases (commercial and free)  
- Academic repositories  
- News for facts *(never as substitute for primary legal authority)*

### 5.5 Evaluating Online Legal Sources

Ask:

1. Is this an **official** source?  
2. Is the judgment / statute **authentic and updated**?  
3. Who published it?  
4. Can I cite it confidently in class / internship?

### Law Office Lens — Citation Discipline

Copying a “nice paragraph” from a random blog into a petition is dangerous. Prefer primary sources. Internet speed is useless without source quality.

### 5.6 Basic Internet Safety for Law Students

- Prefer HTTPS sites for sensitive logins.  
- Do not share OTP / passwords.  
- Beware phishing emails pretending to be from courts/clients.  
- Avoid pirated software (malware + legality issues).  
- Remember: messages and emails can become **evidence**.

### Key Definitions

- **Protocol:** Agreed rules for communication between devices.  
- **TCP/IP:** Core protocol suite of the internet.  
- **Search Engine:** Tool that finds web pages matching queries (Google, Bing).

### Exam Trap

Internet and WWW are **not** identical terms. WWW is part of internet services.

### Quick Check

1. Define Internet.  
2. Differentiate Internet and WWW.  
3. Expand: URL, ISP, IP, DNS.

## 6. Digital Communication

### 6.1 Meaning

**Digital communication** means exchange of information using digital devices and networks — text, audio, video, or documents.

### 6.2 Common Tools in Legal Practice

| Tool | Typical Use | Caution |
|---|---|---|
| Email | Formal client/court/office communication | Tone, attachments, confidentiality |
| Video conferencing | Virtual hearings, meetings | Background, recording consent |
| Instant messaging | Quick updates | Avoid legal advice casually; preserve professionalism |
| Cloud document sharing | Collaborative drafting | Access permissions |
| SMS / official portals | OTP, court alerts, e-filing notices | Verify authenticity |

### 6.3 Email Etiquette for Juniors (High-Value Skill)

Subject line example:  
`Draft Notice for Review — Sharma v. ABC Pvt Ltd — Reply by 5 PM`

Body should be:

- Polite and clear  
- Purpose in first lines  
- Attachments named properly  
- Deadline mentioned  
- Signature with name / enrolment status / contact

### 6.4 Virtual Hearings & Professional Presence

- Stable internet  
- Neutral background  
- Mute when not speaking  
- Documents ready in PDF  
- Know court-specific VC rules

### 6.5 Digital Communication and Evidence Awareness (Intro Level)

Without deep Evidence Act technicalities, remember:

- Screenshots, emails, and chats may be relied upon in disputes.  
- Do not send angry/informal legal conclusions on personal chat casually.  
- Maintain professional records.

### Law Office Lens — One Message, Two Audiences

A WhatsApp update to a client may later be forwarded to opposing counsel or shown in a complaint. Write as if a judge might read it someday.

### Key Definitions

- **Synchronous communication:** Real-time (call, VC).  
- **Asynchronous communication:** Not real-time (email).  
- **Attachment:** File sent with a message.  
- **Encryption (basic idea):** Encoding data to protect it from unauthorised reading.

# Part C — Artificial Intelligence for Law Students

## 7. Introduction to Artificial Intelligence (AI)

### 7.1 Definition (Exam-Ready)

> **Artificial Intelligence** is the branch of computer science that aims to create systems capable of performing tasks that normally require human intelligence — such as understanding language, recognising patterns, solving problems, and making decisions.

### 7.2 Working Idea (Beginner)

AI systems learn from **data** and **rules/models**, then produce outputs such as:

- Predictions  
- Classifications  
- Generated text / images  
- Recommendations  
- Speech recognition

### 7.3 AI vs Automation vs Computer

| Concept | Meaning |
|---|---|
| Computer | General machine for processing instructions |
| Automation | Rule-based repetitive task execution |
| AI | Systems that approximate intelligent behaviour, often learning from data |

Example:  
Mail Merge printing 200 notices = automation.  
A tool summarising 200 judgments and answering questions = AI-assisted work.

### 7.4 Types of AI (Introductory Classification)

1. **Narrow AI (Weak AI):** Good at specific tasks (chatbots, translation, recommendation). This is what we use today.  
2. **General AI (Strong AI):** Human-level general intelligence across domains — **not achieved**.  
3. **Super AI:** Hypothetical beyond-human intelligence — speculative.

### Key Definitions

- **Algorithm:** Step-by-step procedure for solving a problem.  
- **Model:** Learned mathematical representation used by AI systems.  
- **Training Data:** Examples used to teach a model.  
- **Generative AI:** AI that creates new content (text, images, code, audio).

### Exam Trap

“AI always gives correct legal answers.”  
False. AI can hallucinate (invent) citations and misunderstand jurisdiction. Human verification is mandatory.

### Quick Check

1. Define AI.  
2. What is Narrow AI?  
3. Difference between automation and AI with one legal example.

## 8. History of AI (Compact Timeline for Exams)

You do not need memorising every lab experiment. Remember the storyline:

| Era | Milestone | One-line Significance |
|---|---|---|
| 1950 | Alan Turing — “Can machines think?” / Turing Test idea | Philosophical & conceptual foundation |
| 1956 | Dartmouth Conference | Term “Artificial Intelligence” popularised; field begins |
| 1960s–70s | Early expert systems & optimism | Rule-based intelligence attempts |
| AI Winters | Funding/interest drops | Expectations exceeded reality |
| 1990s | Machine learning growth; Deep Blue (1997) beats chess champion | Symbolic win for machine capability |
| 2010s | Deep learning boom; image & speech breakthroughs | AI enters consumer products |
| 2020s | Large Language Models & Generative AI | Chatbots, copilots, legal-tech acceleration |

### Why History Matters to Law Students

Law regulates new technology after social impact appears. Understanding AI’s evolution helps you discuss policy, liability, and ethics with maturity — not hype.

### Law Office Lens

Senior lawyers once said “typewriters are enough.” Then computers became unavoidable. Generative AI is in a similar early-adoption phase: those who learn critically will lead; those who ignore or blindly trust will face risk.

## 9. Applications of AI in the Legal Profession

### 9.1 High-Value Use Cases

1. **Legal Research Assistance:** Finding potentially relevant cases/statutes faster.  
2. **Document Review / Due Diligence:** Scanning large contract sets for clauses and risks.  
3. **Summarisation:** Condensing long judgments into study notes *(verify!).*  
4. **Drafting Assistance:** First drafts of notices, emails, checklists.  
5. **E-Discovery Support:** Helping locate relevant digital evidence in large datasets.  
6. **Predictive / Analytical Tools:** Trends in outcomes (use cautiously; not destiny).  
7. **Translation & Transcription:** Hearing recordings / multilingual support.  
8. **Chatbots for Client Intake:** Basic FAQs and appointment triage in firms.  
9. **Compliance Monitoring:** Tracking regulatory changes for corporate clients.  
10. **Academic Help:** Explaining concepts, generating practice questions for students.

### 9.2 What AI Should *Not* Do Alone

- Final legal advice without advocate review  
- Filing a petition with unverified citations  
- Making ethical decisions  
- Replacing court advocacy judgment and human empathy  
- Handling confidential data on unapproved public tools

### 9.3 Mini Caselets (Real-World Style)

**Caselet A — Research**  
Internship task: “Find recent High Court cases on anticipatory bail in X offence.”  
AI can help brainstorm keywords and summarise publicly available material. You still open authentic databases/court sites and verify ratio and citation.

**Caselet B — Drafting**  
AI generates a legal notice draft in 20 seconds. You check facts, jurisdiction, cause of action, limitation, tone, and office format. AI saved typing time; you supplied legal mind.

**Caselet C — Confidentiality Mistake**  
A junior pastes a sensitive client agreement into a public AI chatbot. That may expose confidential information. Lesson: follow firm policy; prefer approved tools; anonymise where required.

### Key Definitions

- **Legal Tech:** Technology products built for legal workflows.  
- **Hallucination (AI):** Confident but false output, e.g., fake case names.  
- **Human-in-the-loop:** Human reviews/approves AI output before use.

## 10. AI Techniques and Future Scope

### 10.1 Techniques (Beginner-Friendly)

| Technique | Idea in Plain English | Legal-ish Example |
|---|---|---|
| Machine Learning (ML) | Learn patterns from data | Classifying emails as spam vs important |
| Natural Language Processing (NLP) | Understand/process human language | Summarising a judgment |
| Expert Systems | Rules from human experts | Checklist-based eligibility bots |
| Computer Vision | Understand images/video | Reading scanned pages / OCR pipelines |
| Speech Recognition | Voice to text | Transcribing dictation |
| Generative Models | Create new content | Draft outline of a moot memorial |
| Robotics / RPA (related) | Automate repetitive digital tasks | Moving files between systems |

**OCR (Optical Character Recognition):** Converts scanned images of text into editable text — extremely useful for digitising old case papers.

### 10.2 Future Scope (Law + AI)

Likely near-future directions:

1. Deeper integration of AI into e-filing and court administration  
2. Smarter contract analytics for corporate law  
3. Better legal aid chat systems for public access to justice *(with safeguards)*  
4. Stronger regulation around privacy, bias, and accountability  
5. New roles: Legal technologist, AI compliance counsel, prompt-skilled associates  
6. Greater expectation that juniors can use AI tools **responsibly**

### 10.3 Responsible Future Mindset for BA-LLB Students

Learn three muscles together:

1. **Tool skill** — how to use Word/Excel/PPT/AI  
2. **Legal skill** — doctrine, procedure, drafting judgment  
3. **Ethical skill** — confidentiality, honesty, non-discrimination, accountability

### Exam Trap

Writing only “AI will replace lawyers” is shallow. Better answer: AI will automate routine tasks and assist research/drafting, but advocacy, ethics, strategy, and accountability remain human-centred; lawyers who adapt will remain essential.

### Quick Check

1. Explain NLP in one sentence.  
2. What is OCR and why is it useful in courts/chambers?  
3. List four future opportunities created by AI in law.

## 13. Worked Example Set — Hardware & Software in a Legal Matter

### Scenario
Ms. Kavya, a first-year intern, is asked to prepare a bundle for a consumer dispute.

**Step 1 — Input**  
She scans invoices and identity proofs (scanner = input device). She types chronology in Word (keyboard = input).

**Step 2 — Processing**  
CPU processes keystrokes and runs Word/PDF software. RAM holds the open draft and PDF pages currently in use.

**Step 3 — Storage**  
She saves files on SSD in folder `Client_Ramesh_Consumer/Evidence`. She also keeps a cloud backup if office policy allows.

**Step 4 — Output**  
She prints two hard copies for filing and emails a PDF set to the senior advocate.

### What examiner wants in your answer
Not only names of devices, but **flow**: input → process → store → output, with a practical story.

## 14. Deep Dive — Inside the CPU (Still Beginner-Friendly)

When you press the key `A` in a petition draft:

1. Keyboard sends signal.  
2. OS and drivers deliver input to Word.  
3. CPU repeatedly performs the cycle: **Fetch → Decode → Execute**.  
4. Character is stored in the document buffer (RAM).  
5. Display subsystem updates the monitor output.

You do not need engineering detail in BA-LLB exams, but using the words **fetch, decode, execute** shows conceptual clarity.

### ALU vs CU (Memory Hook)

- **ALU** = “Maths & Yes/No logic”  
- **CU** = “Traffic police of instructions”

If a question says “brain of computer,” write **CPU**, then briefly mention ALU + CU + registers.

## 15. Storage Decision Guide for Law Interns

| Need | Prefer | Why |
|---|---|---|
| Daily drafting laptop | SSD | Speed, reliability |
| Large old archives | HDD / office server | Cost-effective capacity |
| Court the same day | PDF on laptop + cloud/email | Access redundancy |
| Highly confidential matter | Encrypted drive / approved office system | Security |
| Sharing with client | Controlled link / email PDF | Avoid editable accidental changes |

### Practical Warning
Pen drives are convenient and dangerous. They get lost in court corridors. Prefer controlled transfers and labelled storage.

## 16. Operating System — Extended Functions with Legal Examples

### 16.1 Process Management
Word, browser, and PDF reader run together during research. OS allocates CPU time so each app gets a turn. When the system freezes, too many processes are competing or one process is stuck.

### 16.2 Memory Management
If RAM is low, OS may use virtual memory (disk space as emergency overflow). Your laptop becomes slow. Lesson: close unused tabs during urgent drafting.

### 16.3 File Management
Creating `Notices`, `Petitions`, `Evidence`, `Orders` folders is OS file management in professional form.

### 16.4 Device Management
Installing a printer driver is OS-level device management. In many chambers, the “printer problem” is actually a driver/permission problem.

### 16.5 Security
User accounts, passwords, firewall settings — first line of digital ethics in practice.

### Classroom Activity (10 minutes)
In pairs, design a folder tree for a criminal bail matter and a corporate notice matter. Compare differences.

## 17. Internet Stack — Simple Mental Model for Law Students

You do not need to become a network engineer. Remember this ladder:

1. You type a **URL** in a browser.  
2. **DNS** finds the matching **IP address**.  
3. Your request travels through your **ISP** over the **Internet**.  
4. Server responds with a webpage (WWW content) using protocols such as **HTTP/HTTPS**.  
5. Browser displays the page.

### Research Quality Ladder (Use this in exams & internships)

**Best:** Official gazette / court website / authenticated database  
**Good:** Reputed academic publisher / recognised commentary (with citation discipline)  
**Weak:** Random blogs, unverifiable WhatsApp forwards, undated PDFs  
**Dangerous:** AI-invented citations presented as real cases

## 20. AI in Law — Benefits, Limits, Ethics (Integrated Answer Framework)

## 22. Full Model Answers (Exam Style)

### Model Answer 1 — Define Computer. Explain characteristics. (5–7 marks)

A computer is an electronic device that accepts data as input, processes it according to programmed instructions, stores data where required, and produces meaningful information as output.

Characteristics:

1. **Speed** — performs operations rapidly; useful in searching large legal databases.  
2. **Accuracy** — gives correct results if instructions and inputs are correct; useful in fee/interest calculation sheets.  
3. **Diligence** — can perform repetitive tasks without fatigue, such as converting many files to PDF.  
4. **Storage capacity** — can store large volumes of case records and precedents.  
5. **Versatility** — supports drafting, research, communication, and presentations.  
6. **Automation** — executes predefined workflows such as mail merge of notices.

In legal practice, computers do not replace professional judgment; they improve efficiency and organisation when used responsibly.

### Model Answer 2 — Operating System: meaning and functions (8–10 marks)

An Operating System (OS) is system software that acts as an interface between the user and computer hardware. It manages resources and provides an environment for application software to run. Examples include Windows, macOS, Linux, Android, and iOS.

Functions:

1. **Process management** — schedules running applications.  
2. **Memory management** — allocates RAM to programs.  
3. **File management** — organises files and folders; controls access.  
4. **Device management** — coordinates printers, scanners, and other peripherals.  
5. **Security** — supports authentication and protection mechanisms.  
6. **User interface** — provides GUI/CLI for interaction.

In a law office, OS skills include disciplined folder structures, safe use of shared systems, timely updates, and confidentiality-preserving habits such as screen lock. Without OS literacy, even good legal drafting becomes poorly managed digitally.

### Model Answer 3 — Applications and limitations of AI in legal profession (10–12 marks)

Artificial Intelligence refers to computing systems designed to perform tasks that typically require human intelligence, such as language understanding, pattern recognition, and decision support. In today’s world, most practical tools are Narrow AI systems specialised for particular tasks.

Applications in legal profession include assisted legal research, contract/document review, judgment summarisation, drafting support, transcription/translation, compliance tracking, and client-intake chatbots. Generative AI can produce first drafts of notices or study outlines quickly, helping students and juniors save time.

However, limitations are serious. AI systems may hallucinate fake citations, miss jurisdiction-specific nuance, or reflect bias present in training data. Confidential client data may be exposed if pasted into unapproved public tools. AI also cannot own professional responsibility; an advocate remains accountable for filings and advice.

Therefore, the correct approach is human-in-the-loop practice: use AI for speed and structure, verify every legal proposition from authentic sources, protect confidentiality, and keep ethics central. AI will reshape legal work, but informed lawyers will remain essential.

### Short Note: RAM vs ROM

- RAM: volatile, read/write, temporary working memory, larger & faster access for apps.  
- ROM: non-volatile, mainly read-only, stores boot instructions.  
- Example: Draft in Word resides in RAM while editing; OS boot code uses ROM/UEFI storage concept.  
- Conclusion: both are primary memory but different roles.

- Meaning of AI
